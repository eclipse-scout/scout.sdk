Scout Editors NLS Editor

Additions to the Java Editor

Scout uses the standard Java Editor as provided by Eclipse. In order to ease the development, some additional features are added the Editor.

Create new Scout Elements

To create new Scout Elements (like Form-Fields, Codes, Columns, Extensions, Form-Handlers, Key-Strokes or Menus) some additional proposals are available in the content assist. With Ctrl+Space the depending on the context, additional templates can be selected to create the corresponding inner-classes. Figure 1 provides an example the content assist proposal in a GroupBox context and Figure 2 does the same for a Table

Figure 1. Proposals to create new entities in a GroupBox
Figure 2. Proposals to create new entities in a Table

When a template is selected, it is possible to customized it by navigating between the different Edit-Groups with the Tab Key (this works exactly like other templates in the Eclipse Editor). With this mechanism you can quickly define the class name, the parent class and other properties. With Enter you will exit the Edit-Mode.

NLS-Entry support

When the carret is inside the String parameter of the TEXTS.get() code, the content assist provide support for the NLS entries. Figure 3 provides an example.

Figure 3. Proposals corresponding to NLS Support.

You can select one of the matching entries (like "DateOfBirth" in the example) and you will see the different translations on the right.

You can select "New text…​" to open the NLS Entry Wizard.
